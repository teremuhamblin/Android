# Code Style

## Java / Kotlin
- CamelCase pour classes et méthodes
- snake_case pour ressources
- Pas de méthodes > 50 lignes
- Documentation KDoc obligatoire

## C/C++
- Indentation 4 espaces
- Pas de macros complexes
- Commentaires Doxygen

## AIDL
- Noms explicites
- Types primitifs privilégiés
