# Dossier RFC

Ce dossier contient toutes les propositions d’évolution majeures du projet.  
Chaque RFC documente un changement d’architecture, un nouveau module ou une décision technique structurante.

## Processus RFC
1. Création d’une issue RFC via GitHub.
2. Discussion et itérations.
3. Rédaction d’un fichier RFC dans ce dossier.
4. Revue par les mainteneurs.
5. Approbation ou rejet.
6. Implémentation si approuvée.

## Numérotation
Les RFC sont numérotées séquentiellement :
- `rfc-0001-...`
- `rfc-0002-...`

## Statuts possibles
- **Proposée**
- **En discussion**
- **Acceptée**
- **Rejetée**
- **Superseded** (remplacée par une autre RFC)
